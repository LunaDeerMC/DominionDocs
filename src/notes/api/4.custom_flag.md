> 自 4.7.0 起，Dominion 开放了自定义 Flag 的功能。旧版 `Flag`、`EnvFlag`、`PriFlag`、`Flags.registerEnvFlag`、`Flags.registerPriFlag` 已标记为过时，新代码应使用 `EnvironmentFlagDefinition`、`PrivilegeFlagDefinition` 和 `FlagRegistry`。

## 简介

新的 Flag API 使用具体的 flag 对象表示权限项，而不是在业务代码中传递 `String flagId`。环境规则使用 `EnvironmentFlagDefinition`，玩家权限使用 `PrivilegeFlagDefinition`。

- `FlagDefinition`：环境 flag 和权限 flag 的共同接口，主要用于 registry、group、storage 等通用逻辑。
- `EnvironmentFlagDefinition`：一个环境 flag，例如 `FlagDefinitions.EXPLODE_TNT`。
- `PrivilegeFlagDefinition`：一个玩家权限 flag，例如 `FlagDefinitions.OPEN_CHEST`。
- `FlagDomain`：flag 所属域，分为 `ENVIRONMENT` 和 `PRIVILEGE`。
- `FlagGroupDefinition`：一组 flag，用于展示组状态和批量开关。
- `FlagGroupState`：组状态，可能为 `ON`、`OFF`、`MIXED`。
- `FlagValueSet`：一组 flag 值，支持读取单个 flag 或计算组状态。
- `FlagRegistry`：flag 与 group 的注册表。
- `FlagDefinitions`：Dominion 内置的新 flag 定义。

内置 flag 推荐直接使用 `FlagDefinitions` 中的对象：

```java
EnvironmentFlagDefinition envFlag = FlagDefinitions.EXPLODE_TNT;
PrivilegeFlagDefinition privilegeFlag = FlagDefinitions.OPEN_CHEST;
```

不要在普通 API 调用中直接传 `"explode_tnt"` 这样的字符串。字符串 id 只用于 JSON 存储、调试输出或少数注册表查询场景。

## 读取和修改 Flag

环境 flag 用于领地环境规则，例如爆炸、火焰、刷怪：

```java
boolean allowTnt = dominion.getEnvFlagValue(FlagDefinitions.EXPLODE_TNT);
dominion.setEnvFlagValue(FlagDefinitions.EXPLODE_TNT, false);
```

访客权限 flag 用于未单独授权玩家的默认权限：

```java
boolean guestCanOpenChest = dominion.getGuestFlagValue(FlagDefinitions.OPEN_CHEST);
dominion.setGuestFlagValue(FlagDefinitions.OPEN_CHEST, true);
```

成员和用户组权限同样使用 `PrivilegeFlagDefinition`：

```java
boolean memberCanFly = member.getFlagValue(FlagDefinitions.FLY_PLAYER);
member.setFlagValue(FlagDefinitions.FLY_PLAYER, true);

boolean groupCanUseDoor = group.getFlagValue(FlagDefinitions.USE_DOOR);
group.setFlagValue(FlagDefinitions.USE_DOOR, true);
```

## 检查 Flag

在事件监听器中可以通过 `DominionAPI` 检查权限。新 API 接收具体的 flag 对象，并在编译期区分环境 flag 与权限 flag：

```java
boolean allowed = DominionAPI.getInstance().checkEnvironmentFlag(
        location,
        FlagDefinitions.EXPLODE_CREEPER
);
```

玩家权限检查示例：

```java
boolean allowed = DominionAPI.getInstance().checkPrivilegeFlag(
        location,
        FlagDefinitions.OPEN_CHEST,
        player
);
```

## 使用 Flag 组

Flag 组用于 UI 展示和批量操作。组内 flag 全部开启时为 `ON`，全部关闭时为 `OFF`，开关不一致时为 `MIXED`。

```java
FlagRegistry registry = FlagRegistry.defaultRegistry();
FlagGroupDefinition explosion = registry.getGroup("environment.explosion");

FlagGroupState state = dominion.getEnvironmentFlags().getGroupState(explosion);
```

批量关闭一个组时，请遍历组内的 flag 对象并逐个持久化：

```java
for (EnvironmentFlagDefinition child : registry.getEnvironmentGroupChildren("environment.explosion")) {
    dominion.setEnvFlagValue(child, false);
}
```

`FlagValueSet#setGroup` 适合在内存中的值集合上做批量修改；如果需要写入数据库，仍应通过 DTO 的 setter 或 Dominion 提供的写入 API 持久化。

## 构造自定义 Flag

自定义 flag 也应定义为静态对象，随后注册到 `FlagRegistry`。环境 flag 使用 `EnvironmentFlagDefinition`：

```java
public final class MyFlags {
    private MyFlags() {
    }

    public static final EnvironmentFlagDefinition ALLOW_RAIN = new EnvironmentFlagDefinition(
            "allow_rain",
            "Allow Rain",
            "Whether weather can change to rain in this dominion.",
            false,
            true,
            Material.SUNFLOWER
    );
}
```

注册 flag：

```java
FlagRegistry.defaultRegistry().registerFlag(MyFlags.ALLOW_RAIN);
```

权限 flag 则使用 `PrivilegeFlagDefinition`：

```java
public static final PrivilegeFlagDefinition USE_CUSTOM_PORTAL = new PrivilegeFlagDefinition(
        "use_custom_portal",
        "Use Custom Portal",
        "Whether players can use custom portals in this dominion.",
        false,
        true,
        Material.ENDER_EYE
);
```

如果需要让 UI 或管理逻辑按组展示，也可以注册自定义分组。组内成员同样使用 flag 对象：

```java
FlagRegistry.defaultRegistry().registerGroup(new FlagGroupDefinition(
        "environment.weather",
        FlagDomain.ENVIRONMENT,
        "Weather",
        "Weather controls",
        List.of(MyFlags.ALLOW_RAIN)
));
```

请在插件启用阶段尽早注册自定义 flag，避免 Dominion 已经初始化缓存或 UI 后再改变注册表。

## 自定义 Flag 的事件逻辑

注册 flag 只会让 Dominion 能保存和读取这个值。具体行为需要插件自己在事件中实现：

```java
@EventHandler
public void onWeatherChange(WeatherChangeEvent event) {
    if (!event.toWeatherState()) {
        return;
    }

    Location location = event.getWorld().getSpawnLocation();
    boolean allowed = DominionAPI.getInstance().checkEnvironmentFlag(location, MyFlags.ALLOW_RAIN);
    if (!allowed) {
        event.setCancelled(true);
    }
}
```

## 旧 API 兼容性

旧版 `EnvFlag`、`PriFlag`、`Flags` 和旧 flag 修改事件仍会暂时保留，用于兼容现有插件和旧 UI/命令，但它们已经标记为过时。新代码应直接依赖 `EnvironmentFlagDefinition`、`PrivilegeFlagDefinition`、`FlagDefinitions`、`FlagRegistry` 和 `FlagValueSet`。
